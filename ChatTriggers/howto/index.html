clickable($text,$clickAction,$clickValue,$hoverText)
Creates clickable text inside of a chat message

  • $text

    The text that shows up in the chat message

  • $clickAction

    Value for that action

    run_command
    open_url

  • $clickValue

    Value for that action command
    URL

  • $hoverText

    Text that shows up when you hover over the clickable chat

Triggers

Creating a trigger

Command
/trigger create [trigger type] [trigger]
Command usage
Create a trigger that trips any events tied to it
Trigger types
chat - gets tripped when a chat message matches the trigger

other - only gets tripped on other trigger events





.

Trigger tags

About
Trigger tags are used inside of triggers to organize them and show special attributes about them

Tags are always surrounded by '<' and '>'

Tags
<list=listName>
The list tag is used to organize triggers. This will help you keep things tidie as you add more and more triggers.
<server=serverIP>
Seperate multiple servers with a comma.
The server that you are currently on only needs to contain the serverIP that you specify, not equal exactly.
use 'SinglePlayer' for offline.
<start> <contain> <end>
These are used to tell ChatTriggers where in the message it is looking for text.










NOTE: Want to make sure you are picking up the right color in your trigger? just add the formatting codes!
If you have one formatting code (eg: &5), if you dont have all of the formatting codes, the trigger WONT get tripped.
you can use '/trigger setting debug' to see all of the color formatting codes the same way the mod sees them

Events

Creating an event

Command
/trigger add [trigger number] [event type] [event arguments]
Command usage
Create an event for when a trigger gets tripped



.

Event types

  • Chat

    Functions

    Shows a client side message in chat

  • Cancel

    Cancels the chat message that triggered

  • Killfeed

    Tags

    Adds to a killfeed in the top left of Minecraft

  • Notify

    Tags

    Animated message in the corner of your screen

  • Trigger

    Trips another trigger, triggering its events


  • Sound

    Play a sound from Minecraft sounds or a custom resource pack


.

Logic events

  • If

    Runs the event block below if arguments are true

  • Else

    Runs the event block below if the arguments of theif is false

  • Choose

    Picks at random one of the event blocks below

  • Wait

    Waits to run the events in the block below

  • For

    Itterates through an array
    For $value : array

  • End

    Used to close an event block
    dont forget this

Strings

Creating a string

Commands
/trigger string create [string name]
/trigger string set [string name] [string]

.

Reading a string

{string<string name>}
This gets replaced with what is stored inside of the string

This can be used in either an trigger or an event

.

Built in strings

  • {me}

    Returns your username

  • {msg}

    Returns the formatted message that tripped the trigger

  • {trigsize}

    returns the number of triggers that you have

  • {notifysize}

    returns the number of notifications there currently are

.

Setting a string with a trigger

{string[string name]}
If this is inside of a trigger, the trigger will attempt to set the string if every thing else in the trigger matches.

This is a very powerful feature and extremely fun to use!


{string[string name]number}
Checks to make sure that if a string is about to be set, it needs to be a certain number of words long.

If the number of words doesnt match the number specified, the string doesnt get set.
Examples
/trigger create {string[name]1} is in {string[game]}

/trigger create Guild > {string[name]}: {string[msg]}

/trigger create {s}{string[name]1} killed

Functions

.

Logic functions

  • {string}.equals($value)

    Returns true if {string} equals $value

  • {string}.contains($value)

    Returns true if {string} contains $value

  • {string}.startsWith($value)

    Returns true if {string} starts with $value

  • {string}.endsWith($value)

    Returns true if {string} ends with $value

.

String functions

  • string.set($string,$value)

    Sets value $value in string $string


    Returns $value

  • {string}.replace($value,$replacement)

    Replaces all of $value in string {string} with $replacement

    Returns modified string

  • {string}.prefix($value)

    Appends prefix $value to string {string}

    Returns modified string

  • {string}.suffix($value)

    Appends suffix $value to string {string}

    Returns modified string

  • {string}.toUpper()

    Changes {string} to be all upper case

    Returns modified string

  • {string}.toLower()

    Changes {string} to be all lower case

    Returns modified string

Commands

/trigger

/t

  • /trigger create [type] [trigger]

    Creates a trigger

    Sets type of trigger to [type]

  • /trigger delete [trigger num]

    Delete a trigger

    Get the trigger numbers from /trigger list

  • /trigger add [trigger num] [event]

    Add an event to the trigger

    Get the trigger numbers from /trigger list

  • /trigger remove [trigger num] [event num]

    Remove an event to the trigger

    Get the trigger and event numbers from /trigger list

/trigger string

/t string

  • /trigger string create [string name]

    Creates a string

  • /trigger string delete [string name]

    Delete a string

  • /trigger set [string name] [value]

    Set the value in a string

/trigger run

/tr

  • /trigger run [trigger]

    Run a trigger
    This can be used as a custom command for trigger pack makers

  • /trigger run

    Prints out all custom defined commands of your imported trigger packs

/trigger settings

/t settings

  • /trigger settings color [color]

    Change the main accent color of ChatTriggers
    Use either the color code (&f) or the name (darkblue)

  • /trigger settings killfeed position [top-left/top-right]

    Change the position of the killfeed

  • /trigger settings debug

    Toggle debug mode
    This will spit out extra info of your triggers and unformatted chat with all of the color codes

  • /trigger settings test
         [onWorldLoad/onWorldFirstLoad]
         [onServerChange/onNewDay]

    Test non chat event types
    Mostly used for debugging

  • /trigger settings beta

    Opt into notifications about new Nightly Builds

Misc commands

  • /t import [import name]

    Import a 1st party trigger pack

  • /t import [import] [import] <...>

    Import multiple 1st party trigger packs at once

  • /t testImport [import name]

    Test an import as a standalone
    Used for debugging

  • /t testImport [import] [import] <...>

    Test multiple imports at the same time
