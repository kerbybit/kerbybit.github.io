Your first triggers



  • Changing chat

    Identifying what we need
    After you decide what chat you want to change, you need to figure out a few things
    - Do I need to search for color codes to make sure I get the right chat?
    - Can I search for the whole chat message or do I need tags to specify where to look?
    - Is the chat dynamic or static?
    • Searching for colors
      If you need to search or store colors, you can use the & color codes to search for them

      As soon as you use one & sign, CT will start searching for all of the color codes so make sure you have them all matching the message

      You can check color codes by doing /trigger settings debug to toggle debug mode on/off
    • Searching for parts of a message
      If you dont need to search the whole message or you need to grab a message that always starts with, contains, or ends with a phrase, use tags to do so

      <start> will search for messages that start with what you specify
      <contain> will search for messages that contain what you specify
      <end> will search for messages that end with what you specify
    • Static chat
      If the chat message is static, then you dont have to do much :)

      Just create your trigger and you are on your way






      static chat
    • Dynamic chat
      If your chat message changes, there are a few things you can try

      First, if you dont need to retain any of the dynamic data, consider using tags from above

      Second, if you do need to retain the dynamic data, you will need to use strings to store this data in

      dynamic chat

  • Using strings
    If you need to retain dynamic data in chat messages, the way to do that is with strings
    Strings allow you to store parts of chat as the chat flows for a truely customized experience

    Lets change the friends join message

    Lets break this into a few parts
    - The message always starts with a name that is one word long
    - The message always ends with " joined." (dont forget the space)
    Looking at these two parameters, we know that we will need one string
    Lets name this string name


    Now, we need our trigger to set this string as chat messages happen

    I know it looks complicated, but lets break it down do its different parts
    - {string[name]} - this will set our name string to whatever comes before ' joined.'
    - {string[name]1} - this will make sure that name is one word long before setting it and running the events
    - {string[name]1} joined. - this is the whole, put together trigger
